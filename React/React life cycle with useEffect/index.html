React Lifecycle & useEffect Guide

Complete classroom notes with examples for teaching React Hooks and Lifecycle.

1. Component Lifecycle in React

Every React component goes through different phases during its life. These phases are called the Component Lifecycle.

Main Lifecycle Phases

  • Mounting → Component is created and shown on screen.
  • Updating → Component re-renders when state or props change.
  • Unmounting → Component is removed from the screen.

Mounting Example

useEffect(() => {
  console.log("Component Mounted")
}, [])

Updating Example

useEffect(() => {
  console.log("Count Updated")
}, [count])

Unmounting Example

useEffect(() => {

  return () => {
    console.log("Component Unmounted")
  }

}, [])

2. Wrong Ways of Fetching Data

Students should avoid these common mistakes while fetching data.

Wrong Way #1 → Fetching directly inside component body

function App() {

  fetch("https://jsonplaceholder.typicode.com/users")

}
Problem: API will call on every render which can cause infinite requests.

Wrong Way #2 → Missing Dependency Array

useEffect(() => {
  fetchData()
})
Problem: useEffect runs after every render.

Wrong Way #3 → Async directly inside useEffect

useEffect(async () => {

})
Problem: React does not expect an async callback directly.

3. useEffect in Detail and Benefits

useEffect is a React Hook used for handling side effects.

What are Side Effects?

  • API Calls
  • Timers
  • Event Listeners
  • Local Storage
  • DOM Manipulation

Basic Syntax

useEffect(() => {

  // side effect code

}, [])

Benefits of useEffect

  • Helps fetch data from APIs
  • Runs code after rendering
  • Makes code cleaner and organized
  • Replaces old lifecycle methods from class components
Important: useEffect runs after the component renders.

4. useEffect Dependency Array

The dependency array controls when useEffect should run.

1. Empty Dependency Array

useEffect(() => {
  console.log("Run only once")
}, [])

Runs only on first render.

2. Dependency Included

useEffect(() => {
  console.log("Count changed")
}, [count])

Runs whenever count changes.

3. Without Dependency Array

useEffect(() => {
  console.log("Runs every render")
})

Runs after every render.

5. Async Function Handling in useEffect

The correct way is to create an async function inside useEffect. 

useEffect(() => {

  const fetchData = async () => {

    let response = await fetch(
      "https://jsonplaceholder.typicode.com/users"
    )

    let data = await response.json()

    console.log(data)
  }

  fetchData()

}, [])
Why? useEffect itself should not be async.

6. useEffect Cleanup

Cleanup functions remove unnecessary processes when a component unmounts.

Common Cleanup Examples

  • Clearing Timers
  • Removing Event Listeners
  • Stopping Subscriptions

Timer Cleanup Example

useEffect(() => {

  let timer = setInterval(() => {
    console.log("Running...")
  }, 1000)


  return () => {
    clearInterval(timer)
  }

}, [])
Benefit: Cleanup prevents memory leaks and improves performance.
