What is State?

State is a built-in React feature used to store dynamic data inside a component.

Definition: State is data that can change over time.

Examples

  • Counter value
  • Input field data
  • Dark mode toggle
  • Cart items
  • Loading status

State Example


import { useState } from "react";

function Counter() {
  const [count, setCount] = useState(0);

  return (
    <div>
      <h1>{count}</h1>

      <button onClick={() => setCount(count + 1)}>
        Increase
      </button>
    </div>
  );
}

export default Counter;

Understanding useState


const [count, setCount] = useState(0);
Part Meaning
count Current state value
setCount Function to update state
useState(0) Initial value

Important Points About State

  • State is mutable
  • State updates trigger re-rendering
  • Every component can have its own state
  • State helps create dynamic UI

Handling Events in React

Events are user interactions with the application.

Event Purpose
onClick Handle button click
onChange Handle input changes
onSubmit Handle form submission
onMouseEnter Handle mouse hover
onKeyDown Handle keyboard press

Button Click Example


function App() {

  function handleClick() {
    alert("Button Clicked");
  }

  return (
    <button onClick={handleClick}>
      Click Me
    </button>
  );
}

export default App;

Input Change Example


import { useState } from "react";

function App() {

  const [name, setName] = useState("");

  return (
    <div>

      <input
        type="text"
        onChange={(e) => setName(e.target.value)}
      />

      <h1>{name}</h1>

    </div>
  );
}

export default App;

useState in Depth

useState is a React Hook that allows functional components to use state. 


const [state, setState] = useState(initialValue);

Different Types of State

String State

const [name, setName] = useState("Majid");

Number State

const [count, setCount] = useState(0);

Boolean State

const [darkMode, setDarkMode] = useState(false);

Array State

const [users, setUsers] = useState([]);

Object State

const [user, setUser] = useState({ name: "Ali", age: 20 });

Updating State

Updating Number

setCount(count + 1);

Updating Object


setUser({
  ...user,
  age: 25
});
Spread operator preserves previous object data.

Rules of useState

Never Update State Directly ❌


count = count + 1;

Correct Way ✅


setCount(count + 1);
  • State updates are asynchronous
  • Components automatically re-render after state updates

State Management

State Management means organizing and managing application data efficiently.

Why Needed?

  • Multiple components need same data
  • Prop drilling problems
  • Large application structure
  • Centralized data management

Types of State Management

Type Description
Local State State inside a single component
Global State Shared across application
Server State Data coming from APIs

Prop Drilling Problem


App → Parent → Child → GrandChild

Passing data through multiple components becomes difficult.

Solutions for State Management

1. Context API


<UserContext.Provider value={user}>

2. Redux

  • Centralized store
  • Predictable state
  • Easy debugging

3. Zustand

  • Lightweight
  • Simple syntax
  • Fast performance
